# 变更说明：新增用户字节级流量累计字段

## 目的

修复用户流量进度统计不准确的问题：

- 逻辑用户跨节点展示时，总配额不应按节点数累加
- 面板按 5 分钟窗口同步流量时，低于 `1 GB` 的增量会因整数取整丢失

本次数据库变更只处理第二个问题：为用户增加字节级累计字段，保留现有 `used_gb` 作为兼容展示字段。

## 范围

- 表：`hysteria_users`
- 新增列：`used_bytes BIGINT UNSIGNED NOT NULL DEFAULT 0`
- 数据回填：当 `used_bytes = 0` 且 `used_gb > 0` 时，将历史整数 GB 值按 `used_gb * 1024^3` 回填

## 执行顺序

1. 执行 SQL 迁移脚本 `database/migrations/2026-06-30_add_hysteria_user_used_bytes.sql`
2. 部署包含后端与前端修复的新版本
3. 验证用户页的流量进度与逻辑用户配额展示

## 回滚方案

1. 回滚应用版本到旧版本
2. 保留 `used_bytes` 列，不建议直接删除

说明：旧版本仍可继续读取 `used_gb`，因此保留新增列即可完成应用层回滚。

## 验证方法

1. 创建或选择一个多节点逻辑用户，确认逻辑用户总配额展示为单份配额，而不是节点数相加
2. 在客户端产生小于 `1 GB` 的流量后，等待一次同步窗口，确认用户页的流量进度不再停留在 `0`
3. 检查 `hysteria_users.used_bytes` 持续增长，且 `used_gb` 与其换算值保持一致

## 风险

- 历史数据只拥有整数 `used_gb` 时，回填后的 `used_bytes` 仍然只能恢复到整 GB 精度，无法找回过去已经丢失的小流量
- 若外部脚本仍直接写入 `used_gb` 而不写 `used_bytes`，新旧字段可能再次出现偏差
